# Git Fork – Working on Someone Else’s Repository

This document explains Git Fork,
why it is used, and how it works on GitHub.

---

## 1. What is Git Fork?

Fork means:
- Creating your own copy of someone else’s repository  
- Repository is copied into your GitHub account

Original repo remains unchanged.

---

## 2. Why Do We Use Fork?

Fork is used when:
- You don’t have write access to a repository
- You want to contribute to open source
- You want to experiment safely

---

## 3. How Fork Works (Concept)

1. Original Repository (Owner)
2. You click **Fork** on GitHub
3. GitHub creates a copy in your account
4. You work on your fork
5. You send changes via Pull Request

---

## 4. Fork vs Clone

| Fork | Clone |
|----|----|
| GitHub feature | Git command |
| Creates new repo | Local copy only |
| Used for contribution | Used for local work |

---

## 5. Forking a Repository (Steps)

1. Open GitHub repository
2. Click **Fork** (top-right)
3. Select your account
4. Fork is created in your profile

---

## 6. Working After Fork

Typical flow:
- Fork repository
- Clone fork to local system
- Create branch
- Make changes
- Commit changes
- Push to fork

---

## 7. Important Notes

✅ Fork keeps original repo safe  
✅ You can sync fork later  
❌ Fork does NOT auto-update  
❌ Direct push to original repo not allowed

## 9. Where Fork is Mostly Used?

- Open source contributions
- Company internal tools
- Learning & experiments

---

## ✅ Summary

- Fork = Copy of repo on GitHub
- Used when no write access
- Safe contribution method
- Works with Pull Requests
